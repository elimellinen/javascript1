Redovisning av kursmoment i kursen Javascript 1

Kmom01

  • Vilken utvecklingsmiljö använder du (berätta om något förändrats sen sist, om du svarat på frågan i tidigare kurser)? Jag använder... Ingen utvecklingsmiljö. Jag har kollar runt på webben för att förstå hur en utvecklingsmiljö för javascript ser ut, men jag tror att just nu i kursen ingen utvecklingsmiljö behövs. Men jag lärde mig att man kan skriva JS i en html-dokument också.
  • Är du bekant med HTML och CSS tillsammans med JavaScript sedan tidigare? Nej, inte alls. Jag visste att JavaScript var ett språk som används mycket, men jag visste absolut ingenting om det.
  • Gick det bra med JSFiddle/CodePen, några funderingar kring verktygen? Ja... Jag förstår inte riktigt vad de är för eller varför vi behöver båda. Kanske den ena kollar JS i förhållande med HTML och CSS och den andra kollar att syntaxen stämmer och allt i JavaScript fungerar bra (jag har läst att JS brukar inte säga till när någonting är fel) Men ändå, det gick bra att validera!
  • Gick det bra att komma i gång med kursmomentet, var det lagom, för litet, för stort, något som var svårt eller saknades? Det var lagom, tror jag. Kanske det vore kul om man skulle få en förklaring på hur språket borde skrivas. T.ex. case sensitive, har en massa olika funktioner som motsvarar "print" eller "return", osv). Men kanske jag säger det bara för jag är förvirrad.
  • Vilken är din TIL för detta kmom? JS kan programmeras client-side, det betyder att jag behöver ingen server för att se vad jag gör i browsern. Koden skickas direkt från min dokument till webbsidan. Och man måste deklarera variabler innan man kan göra någonting, och det finns olika sätt för att göra så.

    • Kmom02

  • Har du programmerat i JavaScript tidigare, eller kan du relatera till andra språk du har programmerat i?
    • Nej, jag har aldrig programmerat i JavaScript och kan delvis relatera till kanske PHP? (den enda jag kan förutom Python, som har ingenting gemensamt med JS tycker jag). Det svåraste med JS är att det inte säger till om när gör fel, det bara... Skriver inte ut. Och då tar det lite tid att förstå var är felet och hur jag kan fixa det. Tur att JSHint finns och konsolen finns.
  • Vad tycker du är det svåraste, mest utmanande i utveckling med HTML, CSS och JavaScript - så här långt?
    • Jag tror att jag inte är riktigt bekväm med JS och HTML, till exempel konstruktionen: variable.innerHTML = "hej". Om jag tittar på det, jag kan inte komma på vilken bit gör vad och hur är det användbart/hur det skiljer sig från andra liknande funktioner. Men har börjat att lyssna på en intressant podcast om JavaScript & JS inbyggda funktioner och jag tror att det har hjälpt mig förstå "logiken".
  • Hur gick det att utföra respektive uppgift, vilken tog mest tid och vilken var mest lärorik?
    • Det gick bra! Den som tog mest tid var att få bollen att flytta runt, men det är för jag försökte förstå själv och inte tittade på guiden/instruktioner, bara genomgången. Det var mest lärorik även om den tog typ 3 hela dagar. Jag tror att ibland även när man går mycket långsammare är det bra att försöka klura ut lösningar själv utan att kolla på instruktionerna. Det var i alla fall bra träning för mig, som har en tendens att ge upp snabbt. Den roligaste var dock att bygga flaggorna med CSS och lära mig hur man gör en triangel.
  • Vilken är din TIL för det här kursmomentet?
    • En triangel i CSS är ingen riktig "triangel" utan en kvadrat med kanterna så tjocka och nära varandra att om 3 är transparent, den fjärde som är kvar ser ut som/är en liksidig triangel.
